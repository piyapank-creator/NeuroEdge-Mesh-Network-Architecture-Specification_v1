# โปรแกรมจำลองเน็ตเวิร์ก NeuroEdge Mesh v1.2

โปรแกรมจำลองเน็ตเวิร์ก BCI (Brain-Computer Interface) แบบ Mesh ที่ครอบคลุมสำหรับวิชา CP352005 Networks โครงการนี้สาธิตการสร้างโมเดล OSI 7 ชั้น พร้อมการประมวลผลสัญญาณประสาท การจัดการโครงสร้างเน็ตเวิร์ก และการกำหนดเส้นทางแบบไดนามิกพร้อมความทนทานต่อความผิดพลาด

## 📋 ภาพรวมโครงการ

NeuroEdge เป็นระบบจำลองที่สร้างแบบจำลองเน็ตเวิร์ก Mesh ที่กระจายตัวของอุปกรณ์เชื่อมต่อประสาท โปรแกรมนี้สาธิต:
- **การใช้งาน OSI Model 7 ชั้น** (Physical → Application)
- **การประมวลผลสัญญาณประสาท** ผ่านอินเทอร์เฟซระหว่างสมองและคอมพิวเตอร์
- **โครงสร้าง Mesh Network** พร้อมการจัดการโหนดแบบไดนามิก
- **การกำหนดเส้นทางแบบอัจฉริยะ** โดยใช้การคำนวณต้นทุน
- **ความทนทานต่อความผิดพลาด** พร้อมกลไกการสำรองข้อมูล
- **การแสดงภาพแบบโต้ตอบ** ของโครงสร้างเน็ตเวิร์กและการไหลของแพ็คเก็ต

### คุณสมบัติหลัก (v1.2)

✨ **ใหม่ใน v1.2:**
- **โหนดสำรอง CLOUD_02** - ขจัดจุดอันตรายเดียวของเน็ตเวิร์กแบบคลาวด์
- **การจำลองความผิดพลาดของโหนด** - คลิกโหนดใดๆ บนผืนผ้าใบเพื่อสลับสถานะ ON/OFF
- **กลไกการสำรองข้อมูลอัตโนมัติ** - การกำหนดเส้นทางใหม่อัตโนมัติเมื่อปลายทางสถานที่ไม่ทำงาน
- **ตัวบ่งชี้ความผิดพลาดด้านสายตา** - โหนดที่ล้มเหลวแสดงด้วยสัญลักษณ์ ✕ และสีที่มืดลง
- **การบันทึกอย่างครอบคลุม** - การติดตามเหตุการณ์และการแจ้งเตือนการสำรองข้อมูล

## 🏗️ สถาปัตยกรรมระบบ

### ชั้นของเน็ตเวิร์ก

| ชั้น | ชื่อ | ความรับผิดชอบ | คลาสหลัก |
|-------|------|-----------------|-------------|
| **L1** | Physical | การสร้างและการส่งสัญญาณประสาท | `PhysicalLayer` |
| **L2** | Data Link (NNAP) | การแก้ไขที่อยู่และเสถียรภาพของลิงก์ | `DataLinkLayer` |
| **L3** | Network (MDR) | การกำหนดเส้นทางและการค้นหาเส้นทาง | `NetworkLayer` |
| **L4** | Transport (NPI) | การตรวจสอบแพ็คเก็ตและกฎความปลอดภัย | `TransportLayer` |
| **L5** | Session (BSP) | การจัดการสถานะการเชื่อมต่อ | `SessionLayer` |
| **L6** | Presentation | การเข้ารหัส/การถอดรหัสความคิด | `PresentationLayer` |
| **L7** | Application | บริการระดับสูงและการส่งข้อความ | `ApplicationLayer` |

### ประเภทของโหนด

- **BCI (USER_001)**: อินเทอร์เฟซระหว่างสมองและคอมพิวเตอร์ - แหล่งสัญญาณประสาท
- **EDGE (EDGE_001, EDGE_002, EDGE_003)**: โหนดขอบเขตสำหรับรีเลย์สัญญาณและการประมวลผล
- **CLOUD (CLOUD_01, CLOUD_02)**: โหนดเก็บข้อมูลและการประมวลผลบนคลาวด์

### โครงสร้างข้อมูลเน็ตเวิร์ก

#### ที่อยู่ NNAP
รูปแบบที่อยู่แบบกำหนดเองที่มี:
- `user_id`: ตัวระบุเฉพาะตัว
- `gps_x, gps_y, gps_z`: พิกัดภูมิศาสตร์
- `signal_type`: Alpha, Beta, Gamma หรือ Theta
- `stability`: ความแรงของสัญญาณ (0.0 - 1.0)

#### แพ็คเก็ตประสาท
บรรจุข้อมูลที่ส่ง:
- `packet_id`: ตัวระบุเฉพาะตัวของแพ็คเก็ต
- `source_address`: ที่อยู่ NNAP ของผู้ส่ง
- `destination_id`: ID ของโหนดปลายทาง
- `command_code`: คำสั่งความคิดที่เข้ารหัส
- `signal_quality`: ตัวชี้วัดความสมบูรณ์ของสัญญาณ
- `emotional_vector`: ข้อมูล Valence และ Arousal
- `hops`: เส้นทางผ่านเน็ตเวิร์ก
- `latency_ms`: เวลาการส่งทั้งหมด

## 🧠 อัลกอริทึมการกำหนดเส้นทาง

### การคำนวณต้นทุน
```
ต้นทุน = (0.5 × ส่วนของเสียง) + (0.3 × ระยะทาง) + (0.2 × (1 - แบตเตอรี่))
```
ที่ไหน:
- **ส่วนของเสียง**: ผลผกผันของเสถียรภาพของสัญญาณ
- **ระยะทาง**: ระยะการทำให้เป็นมาตรฐาน GPS ระหว่างโหนด
- **แบตเตอรี่**: ระดับแบตเตอรี่ปัจจุบันของโหนดรีเลย์

### การเลือกเส้นทาง
1. ระบุโหนด EDGE ทั้งหมดที่อยู่ออนไลน์พร้อมเสถียรภาพ ≥ 0.5
2. คำนวณต้นทุนสำหรับโหนดรีเลย์ตัวเลือกแต่ละตัว
3. เลือกเส้นทางต้นทุนต่ำสุด
4. กลับไปใช้การกำหนดเส้นทางโดยตรงหากไม่มีรีเลย์ที่สามารถใช้ได้

### กลไกการสำรองข้อมูล
- ตรวจสอบสถานะของโหนดปลายทาง
- หากปลายทางไปออฟไลน์ระหว่างการส่ง
- ค้นหาโหนด CLOUD สำรองพร้อมเสถียรภาพสูงสุด
- กำหนดเส้นทางแพ็คเก็ตใหม่โดยอัตโนมัติไปยังสำรอง
- บันทึกเหตุการณ์การสำรองข้อมูลพร้อมเหตุผลและความล่าช้า

## 🎮 วิธีการใช้และการติดตั้ง

### 📦 ข้อกำหนดเบื้องต้นก่อนติดตั้ง

โปรแกรมนี้ต้องการ:
- **Python 3.7 ขึ้นไป** (ตรวจสอบเวอร์ชัน: `python --version`)
- **tkinter** (รวมอยู่ในการติดตั้ง Python มาตรฐาน)
- ระบบปฏิบัติการ Windows, macOS หรือ Linux

### 🔧 ขั้นตอนการติดตั้ง

#### ขั้นตอนที่ 1: ตรวจสอบเวอร์ชัน Python
```bash
python --version
```
ต้องแสดง Python 3.7 หรือสูงกว่า

#### ขั้นตอนที่ 2: ยืนยันว่า tkinter ติดตั้งแล้ว (Windows)
```bash
python -m tkinter
```
หากหน้าต่างทดสอบ tkinter ปรากฏขึ้น แสดงว่าติดตั้งถูกต้อง

**ถ้า tkinter ไม่ติดตั้ง:**
- **Windows (pip):** `pip install tk`
- **Ubuntu/Debian:** `sudo apt-get install python3-tk`
- **Fedora:** `sudo dnf install python3-tkinter`
- **macOS:** `brew install python-tk@3.11` (แทน 3.11 ด้วยเวอร์ชันของคุณ)

#### ขั้นตอนที่ 3: ติดตั้ง Dependencies ที่จำเป็น
```bash
pip install -r requirements.txt
```

หรือติดตั้งแบบแยกรายการ:
```bash
pip install dataclasses  # (if Python < 3.7)
```

**หมายเหตุ:** Python 3.7+ มี `dataclasses` ในตัวแล้ว

#### ขั้นตอนที่ 4: ดาวน์โหลดและตรวจสอบไฟล์
1. ท้ายแน่ใจว่ามีไฟล์ `Neuroedge_simulator.py` ในไดเรกทอรี่
2. คลิกขวา → Properties → ตรวจสอบขนาดไฟล์ควรเป็น ~40+ KB

python Neuroedge_simulator.py

หรือบน Windows คุณสามารถ:

### ✨ ฟีเจอร์แบบโต้ตอบ

#### การสลับสถานะโหนด
- **คลิกโหนดใดๆ** บนผืนผ้าใบเพื่อจำลองสถานะ ON/OFF
- โหนดที่ล้มเหลวจะแสดงด้วยสัญลักษณ์ ✕ และสีที่มืดลง
- ระบบจะกำหนดเส้นทางปริภูมิลอบเวราอบโหนดที่ล้มเหลวโดยอัตโนมัติ

#### การควบคุมการจำลอง
- **ส่งความคิด**: ส่งข้อความประสาทผ่านเน็ตเวิร์ก
- **ดูโครงสร้าง**: การแสดงภาพแบบเรียลไทม์ของเน็ตเวิร์ก
- **ตรวจสอบบันทึก**: การติดตามเหตุการณ์หลายชั้นสำหรับการส่งแต่ละครั้ง

#### คำสั่งที่พร้อมใช้ (ความคิด)
- `Move Forward` - คำสั่งการเคลื่อนไหวแบบง่าย
- `Stop` - หยุดการเคลื่อนไหว
- `Turn Left` / `Turn Right` - นำทาง
- `SOS Emergency` - ประเมินฉุกเฉิน (ทริกเกอร์การส่ง GPS)
- `Share Vision` - การออกอากาศการร่วมมเนสหลายคน
- `Send Message` - การส่งข้อความประสาทโดยตรง

## 🛠️ รายละเอียดทางเทคนิค

### Dependencies
- `tkinter` - GUI และ canvas visualization
- `threading` - Background simulation threads
- `json` - การจัดลำดับแพ็คเก็ต
- `dataclasses` - โครงสร้างข้อมูลที่ปลอดภัยแบบพิมพ์
- `math` - การคำนวณระยะทาง
- `random` - การจำลองการสร้างสัญญาณ
- `datetime` - การแสดงเวลา

### ข้อควรพิจารณาด้านประสิทธิภาพ
- **การจำลองความล่าช้า**: 5-20ms ส่งสัญญาณทางกายภาพ + 10-40ms การประมวลผลแอปพลิเคชัน
- **เกณฑ์เสถียรภาพ**: ≥ 0.5 สำหรับการมีส่วนร่วมของโหนด, < 0.5 ทริกเกอร์การกำหนดเส้นทางใหม่
- **ผลกระทบของแบตเตอรี่**: มีผลต่อการเลือกโหนดรีเลย์ (แบตเตอรี่ที่ต่ำกว่า = ต้นทุนที่สูงขึ้น)

## 🔬 สถานการณ์การจำลอง

### สถานการณ์ที่ 1: ความผิดพลาดของโหนด
1. คลิก CLOUD_01 เพื่อจำลองความผิดพลาด
2. ระบบตรวจจับสถานะนอกออฟไลน์โดยอัตโนมัติ
3. แพ็คเก็ตต่อมากำหนดเส้นทางใหม่ไปยัง CLOUD_02
4. การกู้คืน: คลิกอีกครั้งเพื่อกู้คืนโหนด

### สถานการณ์ที่ 2: การข่มขู่ของสัญญาณ
- โหนดที่มีเสถียรภาพ < 0.5 อพยพโดยอัตโนมัติจากการกำหนดเส้นทาง
- เส้นทางทางเลือกที่เลือกโดยอิงตามโหนดที่เหลือที่เสถียร

### สถานการณ์ที่ 3: การออกอากาศฉุกเฉิน
- ส่งคำสั่ง "SOS Emergency"
- ทริกเกอร์แอปพลิเคชันการบอกตัวตนฉุกเฉิน
- ออกอากาศไปยังโหนดออนไลน์ทั้งหมดพร้อมพิกัด GPS
